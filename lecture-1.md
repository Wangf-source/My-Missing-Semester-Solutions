# 《The Shell》
## 一、Shell 基础
### 1. Shell 
 定义：Shell 是用户与操作系统交互的文字接口，通过命令行输入指令，解析并执行程序。

 核心功能：执行程序、处理输入输出、支持脚本编程（变量、条件、循环等）。

常用 Shell：Bash（Bourne Again SHell）是最广泛使用的 Shell，语法与其他 Shell 类似。
### 2. 终端与提示符
终端：打开终端即可进入 Shell 提示符（如 missing:~$），其中~表示用户主目录，$ 表示普通用户（# 为管理员）。

命令格式：命令 [参数]，参数含空格时需用引号（'My Photos'）或转义符（My\ Photos）。
## 二、文件系统导航
### 1. 路径类型
绝对路径：以 / 开头（如 /home/missing）。

相对路径：相对于当前目录（如 ../bin 表示上级目录的 bin 文件夹）。

常用命令：
pwd：查看当前工作目录。
cd：切换目录（cd .. 到上级目录，cd ~ 到主目录）。
### 2. 文件操作
ls：列出目录内容（ls -l 显示详细信息，含权限、所有者等）。

mkdir：创建文件夹（如 mkdir missing）。

touch：创建空文件。
## 三、文件权限与管理
### 1. 权限解析（ls -l 输出）
示例：drwxr-xr-x 表示：

类型：d（目录），-（文件）。

权限：3 组权限（所有者、用户组、其他用户），每组 r（读）、w（写）、x（执行），- 表示无权限。
### 2. 权限修改
chmod：修改文件权限。
语法：chmod [权限模式] 文件名，如 chmod +x semester（添加执行权限）。

权限模式：数字法（755 对应 rwxr-xr-x）或符号法（u+x 给所有者添加执行权限）。
### 四、输入输出重定向与管道
1. 重定向>：输出重定向到文件（覆盖），如 echo hello > hello.txt。>>：追加内容到文件。
<：输入重定向，如 cat < hello.txt。
2. 管道（|）
连接多个程序的输入输出，如 ls -l / | tail -n 1（将 ls 的输出作为 tail 的输入）。
### 五、高级技巧
1. sudo 与权限问题
sudo：以管理员权限执行命令，解决 Permission denied 错误。
注意：重定向操作由 Shell 处理，需用 tee 配合 sudo，如 echo 3 | sudo tee brightness（避免 Shell 权限不足）。
2. shebang 行
脚本第一行 #!/bin/sh 告知系统使用 sh 解释器执行脚本，确保正确解析。
